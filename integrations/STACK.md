# Recommended Tool and Plugin Stack

Install and connect tools only when there is a defined owner, purpose, minimum permission set and review date.

| Capability | Recommended integration | Use |
|---|---|---|
| Source control | GitHub | Agent playbooks, templates, changes and auditability |
| Creative design | Figma | Premium templates, campaign systems and collaborative review |
| Brand assets | Google Drive or approved asset system | Photos, video, logos and approved source material |
| Planning | Notion or existing PM system | Calendar, briefs, knowledge and campaign status |
| Team approvals | Slack or Teams | Approval queues, alerts and daily summaries |
| Email | Gmail or Outlook Email | Draft campaigns and approved outreach |
| Scheduling | Google or Outlook Calendar | Production days, launches and owner reviews |
| Research | Live web research | Current trends, competitors, SEO and evidence |
| Imagery | Image generation | Concepts, backgrounds and approved campaign assets |
| Analytics | Existing website/ad/CRM reporting | Source-to-booking and source-to-revenue measurement |

## Priority order

1. GitHub foundation.
2. Existing source-of-truth analytics and CRM reporting.
3. Figma plus approved brand/photo library.
4. Team approval channel.
5. Email/calendar connectors.
6. Additional systems only when they remove a verified bottleneck.

## Connection checklist

- Business owner approved.
- Data owner named.
- Minimum scopes selected.
- PHI exposure reviewed.
- Write actions approval-gated.
- Test workspace used first.
- Disconnect/incident owner named.
- 90-day access review scheduled.

## Social platform note

Native publishing integrations are optional. Drafting, production planning and analytics can begin without them. Direct publishing should be added only after approval roles, account security and rollback procedures are tested.
